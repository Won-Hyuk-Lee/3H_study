# 1. 개발 환경 및 도구 기초 (Git, IDE, AI)

## 1. Git 기초 (10~15분 완성)
Git은 **버전 관리 시스템**입니다. 게임의 세이브 포인트와 같습니다.

### 핵심 개념
- **Repository (저장소)**: 프로젝트 파일과 변경 이력이 저장되는 곳.
- **Commit (커밋)**: 현재 상태를 저장하는 행위 (세이브).
- **Push (푸시)**: 내 컴퓨터의 커밋들을 원격 저장소(GitHub)로 업로드.
- **Pull (풀)**: 원격 저장소의 변경 사항을 내 컴퓨터로 가져오기.

### 필수 워크플로우
1. **작업 전**: `git pull` (동료의 작업물 가져오기)
2. **작업 후**:
   - `git add .` (변경사항 장전)
   - `git commit -m "작업 내용 요약"` (로컬에 저장)
   - `git push` (서버에 업로드)

## 2. IDE 활용법 (Visual Studio / Rider)
효율적인 코딩을 위한 필수 단축키와 기능입니다.

### 필수 단축키 (Visual Studio 기준)
- **정의로 이동**: `F12` (변수나 함수가 선언된 곳으로 이동)
- **전체 검색**: `Ctrl + Shift + F`
- **빠른 수정/리팩토링**: `Ctrl + .` (빨간줄 떴을 때 해결책 보기, 네임스페이스 추가 등)
- **빌드**: `Ctrl + Shift + B`
- **디버깅 시작**: `F5`

### 디버깅 기초
- **Breakpoint (중단점)**: 코드 라인 왼쪽을 클릭하여 빨간 점 생성. 실행 중 해당 줄에서 멈춤.
- **조사식 (Watch)**: 멈춘 상태에서 변수 값을 확인하거나 수정해볼 수 있음.

## 3. AI 툴 활용법 (ChatGPT, Copilot, Claude)
AI는 훌륭한 **Pair Programmer**입니다.

### 좋은 질문(Prompt) 예시
- **코드 설명**: "이 함수가 하는 역할을 한 줄씩 해석해줘."
- **버그 수정**: "이 코드에서 NullReferenceException이 발생하는데 원인이 뭘까?"
- **리팩토링**: "이 코드를 좀 더 읽기 좋고 성능 효율적으로 바꿔줘."
- **유니티 특화**: "유니티에서 오브젝트를 회전시키는 가장 효율적인 방법은 뭐야?"

### 주의사항
- AI가 짠 코드는 **반드시 검증**해야 합니다. (실행해보고, 이해해야 함)
- 보안이 중요한 정보(API Key, 비밀번호)는 AI에게 입력하지 마세요.
